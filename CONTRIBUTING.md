# Contributing to the Chamorro Language AI Initiative

Thank you for your interest in contributing. This project depends on collaboration from developers, linguists, educators, cultural practitioners, and community members. All contributions are welcome as long as they align with the mission of preserving and strengthening the Chamorro language.

---

## 1. How to Contribute

### 1.1 Reporting Issues
If you find a bug, inconsistency, or have a suggestion:
- Open an Issue in the repository.
- Provide clear steps to reproduce (if applicable).
- Include relevant context or references.

### 1.2 Submitting Pull Requests
1. Fork the repository.  
2. Create a new branch for your contribution.  
3. Make your changes with clear commit messages.  
4. Submit a Pull Request with:
   - A summary of changes  
   - Any relevant issue numbers  
   - Notes for reviewers  

### 1.3 Types of Contributions
- Data scraping scripts  
- Data cleaning and annotation  
- Documentation improvements  
- Model training scripts  
- Evaluation tools  
- API development  
- Web or mobile UI components  
- Cultural or linguistic review  

---

## 2. Code Standards

### 2.1 Style
- Use clear, readable code.
- Follow Python PEP8 for Python code.
- Include comments where logic may not be obvious.

### 2.2 Testing
- Include tests for new features when applicable.
- Ensure existing tests pass before submitting a PR.

---

## 3. Data and Cultural Sensitivity

This project involves a living Indigenous language. Contributors must:
- Respect cultural boundaries.
- Avoid scraping or uploading copyrighted or sensitive materials.
- Follow all licensing requirements.
- Avoid including sacred, restricted, or inappropriate content.

When in doubt, ask before proceeding.

---

## 4. Community Expectations

All contributors must follow the project's Code of Conduct.  
We aim to maintain a respectful, collaborative, and inclusive environment.
